This plan instance carries 8 stages: research → concept-pick → script →
velocity → storyboard → asset → edit → compose → publish. Each stage
declares its required artifacts in/out, the wavelet/brandwork tools it
may call, and the validators that gate transition to the next stage.

Sequencing is by document order — the runtime executes stages
top-to-bottom unless a `:BLOCKER:` property declares an explicit
cross-edge. Within a stage, parallelism is allowed (shot generation
fans out across scenes; music + dialogue run concurrently with
shot-gen once storyboard lands).

The =:ASSIGNED_AGENT: wavelet-director= property on each stage points
to the agent file at =../agents/wavelet-director.org=. Future plan
variants (=wavelet-ugc.org=, =wavelet-motion-graphics.org=) reuse the
same agent with concept-specific stage tweaks.

GAP notes flag validator KINDs not yet in w.org's registry — those
need to land before this plan can lint clean against a strict
glossary.

* Brief                                                  :input:9-line:
  :PROPERTIES:
  :ID:        brief
  :END:

  The user's input — a 9-slot ad creative spec (PRODUCT / AUDIENCE /
  INSIGHT / PROMISE / PROOF / TONE / MUSIC / CALL / RUNTIME). Long-form
  prose briefs get distilled into this shape during the research stage.

* TODO Stage 2 — concept-pick                  :stage:wavelet-director:
  :PROPERTIES:
  :ID:               concept-pick
  :ASSIGNED_AGENT:   wavelet-director
  :ARTIFACTS_IN:     brief.md strategy.md
  :ARTIFACTS_OUT:    strategy.md
  :END:

  Pick exactly one of the four creative concepts (Organic / Direct
  Response / Animated / Cinematic). Decision criteria:
  1. What concept do the brand's actual published ads live in?
     (=brandwork brand.ads= from stage 1). Match that.
  2. Medium-implied: talking-head → organic; paint/abstract →
     animated; polished real-world → cinematic; hook+CTA-shaped →
     direct response.
  3. Don't pick by genre/brand-vertical priors.

  Record the picked concept in =strategy.md= under a `Concept`
  heading. This stage is intentionally lightweight — it's a
  decision, not a build — but it gates the discipline-loadout
  step the runtime performs before downstream stages.

* TODO Stage 6 — asset                         :stage:wavelet-director:
  :PROPERTIES:
  :ID:               asset
  :ASSIGNED_AGENT:   wavelet-director
  :ARTIFACTS_IN:     storyboard.json velocity.json
  :ARTIFACTS_OUT:    music/track.wav shots/
  :END:

  Generate the paid assets. Three sub-paths run in parallel once
  storyboard lands:
  1. Music gen via =wavelet_music_gen= (Lyria 3 Pro / Lyria 3 Clip).
  2. Dialogue TTS via =wavelet_dialogue_tts= (Kokoro / ElevenLabs).
  3. Per-shot generation via =wavelet_shot_txt2vid= (Fal Veo 3.1 or
     Veo 3.1 ref-to-video) — one call per shot, parallelized.

  Reference stills (character + product refs) for ref-to-video
  shots are generated by =wavelet_shot_still= (Nano Banana 3
  default; Flux Schnell opt-in for batch variants). Cost-gated by
  =--max-cost= per call; total spend gated by stage budget.

  The =frame_judge= tool runs per-shot after generation to catch
  failed clips before they reach compose (saves a re-roll cycle).

* TODO Stage 8 — compose                       :stage:wavelet-director:
  :PROPERTIES:
  :ID:               compose
  :ASSIGNED_AGENT:   wavelet-director
  :ARTIFACTS_IN:     cuts.edl shots/ refs/
  :ARTIFACTS_OUT:    commercial.html scenes/
  :END:

  Author per-scene HTML files in =scenes/= — full HTML/CSS freedom,
  no template library. Each scene contains an inline =<video src=
  "../shots/shot-N.mp4" autoplay muted playsinline>= element (NOT
  =data-video-bg= on the top-level section — that's a hallucinated
  attribute the renderer doesn't honor; lint rule from wb-a2z2
  catches it). HTML overlays for text + brand wordmark + CTA layer
  on top via absolute positioning above the video element.

  Then assemble a single top-level =commercial.html= manifest
  listing every scene + audio cue. HTML is the ONLY composition
  input =wavelet render= accepts.

* Open questions surfaced by this plan dry-run

  Meta-notes for follow-up wb-nlln sub-issues. Not part of the plan.

** Wavelet-specific validator KINDs missing from w.org registry
   The Validator KIND registry in w.org currently lists 6 kinds.
   Wavelet needs 7 additional kinds:
   - =brief_check_passes= (wavelet brief check exit 0)
   - =brandwork_research_done= (trace-based check, impl exists)
   - =screenplay_duration_fits= (wavelet screenplay validate exit 0)
   - =continuity_check_zero_errors= (zero findings in JSON output)
   - =cost_below_usd= (sum trace cost vs =:ARG_MAX:=)
   - =comp_verify_passes= (wavelet verify exit 0)
   - =wavelet_lint_passes= (wavelet lint exit 0, optionally --mp4)
   File a sub-issue: "wb-nlln: extend w.org Validator KIND registry
   with wavelet-specific kinds (brief_check_passes,
   brandwork_research_done, screenplay_duration_fits,
   continuity_check_zero_errors, cost_below_usd, comp_verify_passes,
   wavelet_lint_passes)".

** Plan-variant per concept
   Once concept-pick dispatch lands (wb-q4dk conditional sub-tree
   loading), the runtime can select between =wavelet-commercial.org=
   (this file — general default), =wavelet-ugc.org= (organic-
   register variant with character-define + 9:16 selfie defaults),
   and =wavelet-motion-graphics.org= (animated-register variant
   with no character refs but text-in-Veo gen). Multiple plans
   share the same agent file.

** Per-stage parallelism
   Stage 6 (asset) has three sub-paths that should fan out
   concurrently (music, dialogue, shots). Stages 3-5 (script,
   velocity, storyboard) could also overlap with the start of
   research if the brief is already known. Today this org file
   encodes sequential execution by document order; explicit
   parallel-branch semantics need wb-lrul (parallel task scheduling)
   to land before the runtime can exploit them.

** Concept-pick gating
   Stage 2 (concept-pick) writes the picked concept to =strategy.md=
   but the runtime doesn't read it back to switch downstream stage
   behavior. That dispatch logic is wb-q4dk's job. Until it lands,
   the agent still picks the concept and self-disciplines per the
   SKILL.md guidance — the worktree just doesn't auto-load a
   concept-specific sub-tree.
